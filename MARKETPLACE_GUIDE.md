## Getting Started

#### 3. Filter Resources

1. Click on a resource to view details

## Resource Types

### AAC Boards
Augmentative and Alternative Communication boards for speech therapy. These boards help children communicate using symbols, images, and text.

**Best Practices:**
- Include clear, recognizable symbols
- Use appropriate grid sizes (2√ó2, 3√ó3, 4√ó4, or custom)
- Add multilingual labels when possible
- Test with actual users before uploading

### Visual Schedules
Step-by-step visual schedules for daily routines. Help children understand and follow sequences of activities.

**Best Practices:**
- Use clear, sequential steps
- Include visual symbols for each step
- Keep schedules age-appropriate
- Test the flow before uploading

### Matching Activities
Matching games and activities for cognitive development. Help children develop pattern recognition and association skills.

**Best Practices:**
- Create clear matching pairs
- Use high-quality images
- Vary difficulty levels
- Include instructions

### Sorting Activities
Sorting and categorization activities for learning. Help children develop classification and organizational skills.

**Best Practices:**
- Define clear categories
- Use distinct visual differences
- Provide examples
- Include answer keys

### Custom Activities
Any other therapy-related activities and resources.

**Best Practices:**
- Clearly describe the activity
- Include usage instructions
- Specify target age group
- Provide learning objectives

## Technical Details

### Resource Data Format

Resources are stored as JSON objects. Here's the structure:

```json
{
  "id": "uuid",
  "title": "My AAC Board",
  "description": "A communication board for daily routines",
  "type": "aac_board",
  "category_id": "uuid",
  "price": 0,
  "creator_id": "uuid",
  "file_data": {
    "gridSize": "3x3",
    "cells": [
      {
        "id": "cell-1",
        "symbolId": "symbol-eat",
        "label": "Eat",
        "audioText": "I want to eat"
      }
    ]
  },
  "preview_image": "https://example.com/preview.jpg",
  "downloads_count": 42,
  "rating_avg": 4.5,
  "rating_count": 10,
  "is_published": true,
  "created_at": "2025-01-15T10:00:00Z",
  "updated_at": "2025-01-15T10:00:00Z"
}
```

### Database Schema

#### Resources Table
- `id`: Unique identifier
- `title`: Resource name
- `description`: Detailed description
- `type`: Resource type (aac_board, visual_schedule, etc.)
- `category_id`: Category reference
- `price`: Price in rupees (0 for free)
- `creator_id`: User who created the resource
- `file_data`: JSONB content
- `preview_image`: Image URL
- `downloads_count`: Number of downloads
- `rating_avg`: Average rating (0-5)
- `rating_count`: Number of ratings
- `is_published`: Publication status
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp

#### Resource Categories Table
- `id`: Unique identifier
- `name`: Category name
- `description`: Category description
- `icon`: Emoji or icon
- `created_at`: Creation timestamp

#### Resource Downloads Table
- `id`: Unique identifier
- `resource_id`: Resource reference
- `user_id`: User who downloaded
- `downloaded_at`: Download timestamp

#### Resource Ratings Table
- `id`: Unique identifier
- `resource_id`: Resource reference
- `user_id`: User who rated
- `rating`: Star rating (1-5)
- `review`: Optional text review
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp

#### Resource Purchases Table
- `id`: Unique identifier
- `resource_id`: Resource reference
- `user_id`: User who purchased
- `amount`: Purchase amount
- `purchased_at`: Purchase timestamp

### Security & Permissions

#### Row Level Security (RLS)
- **Resources**: Public can read published resources, authenticated users can create, users can update/delete own resources
- **Downloads**: Users can read own downloads, authenticated users can create downloads
- **Ratings**: Public can read ratings, authenticated users can create/update own ratings
- **Purchases**: Users can read own purchases, authenticated users can create purchases

#### Access Control
- Anyone can browse and view resources
- Authentication required to download, rate, or upload
- Users can only edit/delete their own resources
- Users can only rate resources they've downloaded
- Download tracking prevents duplicate downloads

## Troubleshooting

### Common Issues

#### "Failed to upload resource"
- **Cause**: Invalid JSON in resource data or missing required fields
- **Solution**: Validate your JSON format, ensure title and type are provided

#### "Failed to download resource"
- **Cause**: Not logged in or network error
- **Solution**: Log in first, check internet connection, try again

#### "Cannot rate this resource"
- **Cause**: Haven't downloaded the resource yet
- **Solution**: Download or purchase the resource first

#### "Resource not found"
- **Cause**: Resource was deleted or URL is incorrect
- **Solution**: Return to marketplace and search again

#### "Permission denied"
- **Cause**: Trying to edit/delete someone else's resource
- **Solution**: You can only edit your own resources

### Getting Help

If you encounter issues:
1. Check this guide for solutions
2. Verify you're logged in
3. Try refreshing the page
4. Clear browser cache
5. Contact support if issues persist

## API Reference

### Get Resources
```typescript
getResources(filters?: ResourceFilters): Promise<Resource[]>
```

Fetch resources with optional filters:
- `type`: Resource type
- `category_id`: Category filter
- `price_type`: 'all', 'free', or 'paid'
- `min_rating`: Minimum rating threshold
- `search`: Search query
- `sort_by`: 'newest', 'popular', or 'highest_rated'

### Get Resource by ID
```typescript
getResourceById(id: string): Promise<Resource | null>
```

Fetch a single resource with full details including creator and category information.

### Create Resource
```typescript
createResource(resource: Omit<Resource, 'id' | 'created_at' | 'updated_at' | 'downloads_count' | 'rating_avg' | 'rating_count'>): Promise<Resource | null>
```

Upload a new resource. Requires authentication.

### Download Resource
```typescript
downloadResource(resourceId: string, userId: string): Promise<boolean>
```

Track a resource download. Creates download record and increments download count.

### Rate Resource
```typescript
rateResource(resourceId: string, userId: string, rating: number, review?: string): Promise<ResourceRating | null>
```

Add or update a rating for a resource. Requires prior download.

### Get My Resources
```typescript
getMyResources(userId: string): Promise<Resource[]>
```

Fetch all resources uploaded by the user.

### Get My Downloads
```typescript
getMyDownloads(userId: string): Promise<ResourceDownload[]>
```

Fetch all resources downloaded by the user.
